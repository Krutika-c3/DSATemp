Find the Repeating and Missing Number

Problem Statement: You are given a read-only array of N integers where each integer is in the range [1, N]. It is guaranteed that one number appears twice (repeating) and one number is missing. Your task is to find these two numbers.

Approaches

1. Brute Force Approach

Logic:

  • Iterate through every number from 1 to N (let's call the current number i).
  • For each i, iterate through the entire input array to count its occurrences.
  • If the count of i is 2, then i is the repeating number.
  • If the count of i is 0, then i is the missing number.
  • Stop once both are found.

Algorithm:

  1. Initialize repeating = -1 and missing = -1.
  2. Loop i from 1 to N:

  • Initialize count = 0.

  • Inner loop j from 0 to N-1:

  • If arr[j] == i, increment count.

  • If count == 2, set repeating = i.

  • If count == 0, set missing = i.

  • If both are found, break the loop.

  1. Return {repeating, missing}.

Complexity:

  • Time Complexity: O(N^2) (For each number 1 to N, we traverse the array).
  • Space Complexity: O(1) (No extra space used).

Java Code:

public class Solution {
    public static int[] findMissingRepeatingNumbers(int[] a) {
        int n = a.length;
        int repeating = -1, missing = -1;

        for (int i = 1; i <= n; i++) {
            int count = 0;
            for (int j = 0; j < n; j++) {
                if (a[j] == i) {
                    count++;
                }
            }

            if (count == 2) repeating = i;
            else if (count == 0) missing = i;

            if (repeating != -1 && missing != -1) break;
        }
        return new int[]{repeating, missing};
    }
}

2. Better Approach (Using Hashing)

Logic:

  • Instead of counting frequencies in nested loops, use a frequency array (hash array) to store counts.
  • The size of the hash array will be N + 1.
  • Iterate the input array once to populate the hash.
  • Iterate the hash array from 1 to N to identify the numbers with frequency 2 and 0.

Algorithm:

  1. Create hash array of size N+1 initialized to 0.
  2. Iterate through the input array a:
  • Increment hash[a[i]].
  1. Iterate from 1 to N:
  • If hash[i] == 2, repeating = i.
  • If hash[i] == 0, missing = i.
  1. Return result.

Complexity:

  • Time Complexity: O(2N) (One pass to fill hash, one pass to check hash).
  • Space Complexity: O(N) (For the hash array).

Java Code:

public class Solution {
    public static int[] findMissingRepeatingNumbers(int[] a) {
        int n = a.length;
        int[] hash = new int[n + 1];

        // Update hash array
        for (int i = 0; i < n; i++) {
            hash[a[i]]++;
        }

        int repeating = -1, missing = -1;
        // Check frequencies
        for (int i = 1; i <= n; i++) {
            if (hash[i] == 2) repeating = i;
            else if (hash[i] == 0) missing = i;

            if (repeating != -1 && missing != -1) break;
        }
        return new int[]{repeating, missing};
    }
}

3. Optimal Approach 1 (Mathematical Equations)

Logic:
We can solve this system of equations without extra space using the properties of Sum and Sum of Squares.
Let X be the repeating number and Y be the missing number.

  1. Sum Difference:
  • Let S = sum of elements in the array.
  • Let Sn = sum of natural numbers from 1 to N = N(N+1)/2.
  • S - Sn = X - Y (Equation 1).
  1. Square Sum Difference:
  • Let S2 = sum of squares of elements in the array.
  • Let S2n = sum of squares of 1 to N = N(N+1)(2N+1)/6.
  • S2 - S2n = X^2 - Y^2.
  • Since X^2 - Y^2 = (X - Y)(X + Y), we can rewrite this as:
  • S2 - S2n = (X - Y)(X + Y) (Equation 2).
  1. Solving:

  • From Eq 1, we know val1 = X - Y.

  • From Eq 2, val2 = X^2 - Y^2.

  • Therefore, X + Y = val2 / val1. Let this be val3.

  • Now we have:

  • X - Y = val1

  • X + Y = val3

  • Adding them: 2X = val1 + val3 -> X = (val1 + val3) / 2.

  • Y = X - val1.

Algorithm:

  1. Calculate Sn and S2n using formulas.
  2. Iterate array to calculate S and S2.
  3. Compute val1 = S - Sn.
  4. Compute val2 = S2 - S2n.
  5. Compute val3 = val2 / val1.
  6. X = (val1 + val3) / 2.
  7. Y = X - val1.

Complexity:

  • Time Complexity: O(N) (Single pass).
  • Space Complexity: O(1).

Java Code:

public class Solution {
    public static int[] findMissingRepeatingNumbers(int[] a) {
        long n = a.length; // Use long to prevent overflow
        
        // Sum of N and Sum of Squares of N
        long SN = (n * (n + 1)) / 2;
        long S2N = (n * (n + 1) * (2 * n + 1)) / 6;
        
        long S = 0, S2 = 0;
        for (int i = 0; i < n; i++) {
            S += a[i];
            S2 += (long)a[i] * (long)a[i];
        }
        
        long val1 = S - SN; // X - Y
        long val2 = S2 - S2N; // X^2 - Y^2
        
        long val3 = val2 / val1; // X + Y
        
        long x = (val1 + val3) / 2;
        long y = x - val1;
        
        return new int[]{(int)x, (int)y};
    }
}

4. Optimal Approach 2 (XOR Method)

Logic:
This approach relies on bitwise XOR properties (A ^ A = 0, A ^ 0 = A).

  1. XOR Everything: XOR all array elements and all numbers from 1 to N.
  • Result xr = (arr[0]^...^arr[n-1]) ^ (1^...^N).
  • Everything cancels out except X (repeating) and Y (missing).
  • So, xr = X ^ Y.
  1. Find Differentiating Bit:
  • Since X != Y, xr will have at least one bit set (value 1). This bit represents a position where X and Y differ.
  • Find the rightmost set bit: bitNo = xr & ~(xr - 1).
  1. Bucketing:
  • Separate all numbers (both from array and 1..N) into two buckets based on whether that specific bit is 0 or 1.
  • Zero Bucket: XOR of all numbers where that bit is 0.
  • One Bucket: XOR of all numbers where that bit is 1.
  • Eventually, X will end up in one bucket and Y in the other (because they differ at that bit). All other numbers cancel out.
  1. Identification:
  • We now have two numbers, but we don't know which is missing and which is repeating.
  • Traverse the array once more to see which one appears zero times (missing) vs twice (repeating).

Algorithm:

  1. Calculate xr = 0. XOR all inputs and 1..N.
  2. Find rightmost set bit: number = xr & ~(xr - 1).
  3. Initialize zero = 0, one = 0.
  4. Loop through array and 1..N:
  • If element's bit is set, one = one ^ element.
  • Else, zero = zero ^ element.
  1. Check count of zero in array to see if it is the repeating number. If count == 2, repeating = zero, else repeating = one.

Complexity:

  • Time Complexity: O(N) (Pass 1 to find XOR, Pass 2 to bucket, Pass 3 to verify).
  • Space Complexity: O(1).

Java Code:

public class Solution {
    public static int[] findMissingRepeatingNumbers(int[] a) {
        int n = a.length;
        int xr = 0;

        // Step 1: Find XOR of X and Y
        for (int i = 0; i < n; i++) {
            xr = xr ^ a[i];
            xr = xr ^ (i + 1);
        }

        // Step 2: Find the differentiating bit number
        int number = (xr & ~(xr - 1));

        // Step 3: Group elements into two buckets
        int zero = 0;
        int one = 0;

        for (int i = 0; i < n; i++) {
            // Check array elements
            if ((a[i] & number) != 0) {
                one = one ^ a[i];
            } else {
                zero = zero ^ a[i];
            }
            
            // Check 1 to N
            if (((i + 1) & number) != 0) {
                one = one ^ (i + 1);
            } else {
                zero = zero ^ (i + 1);
            }
        }

        // Step 4: Verify which is repeating
        int cnt = 0;
        for (int i = 0; i < n; i++) {
            if (a[i] == zero) cnt++;
        }

        if (cnt == 2) return new int[]{zero, one};
        return new int[]{one, zero};
    }
}
